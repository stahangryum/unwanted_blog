Problem I

강화유리와 유리를 구분할 수 있는 유리 장인이 있다.

이 유리장인은 80%의 확률로 강화유리를 고른다.

총 10명의 참가자가 있고 이 참가자들은 유리와 강화유리의 조합으로 이루어진 징검다리를 5번 연속으로 건너야 한다.

아래의 경우에 참가자들은 총 몇명이 살아남는가? 1000번의 시뮬레이션을 하여 추정하라.

- 첫 번째 경우

일반인1 - 일반인2 - $\dots$ - 일반인9 - 유리장인||(유리, 강화유리)

- 두 번째 경우

유리장인 - 일반인1 - 일반인2 - $\dots$ - 일반인9||(유리, 강화유리)

Solution II

Outline

총 살아남는 사람 수 = 5 + 장인의 성공 횟수 + 일반인의 성공 횟수 (단, 장인이 4번 또는 5번 성공하지 않았을 때)

일반인의 성공 횟수 = (5-장인의 성공횟수-1) 의 기회 중에서 성공 횟수

5 + 장인이 5번 성공 -> 10

5 + 장인이 4번 성공 -> 10

5 + 장인이 3번 성공 -> 8 + 1번의 기회 중에서 성공 횟수

5 + 장인이 2번 성공 -> 7 + 2번의 기회 중에서 성공 횟수

5 + 장인이 1번 성공 -> 6 + 3번의 기회 중에서 성공 횟수

  • 장인의 성공 횟수

# 변수들의 모음

    toss = function(p) rbinom(n=1, size=1, prob=p) %>% as.logical

reset = function(){
    toss_rslt <<- NA
    surv <<- 10
    stage <<- 0
    player <<- arr[surv]
}

record = function() return(list(toss_rslt=toss_rslt, surv=surv, stage=stage, player=player))

move = function(){
    prob <<- 0.5 + (player=='A')*0.3
    toss_rslt <<- toss(prob)
    if(toss_rslt == FALSE) surv <<- surv - 1
    stage <<- stage + 1
    player <<- arr[surv]
}

gogo = function() for (i in 1:5) move()

gogo_history = function(){
    rslt = as_tibble(record())
    rslt
    for(i in 1:5){
        gogo()
        rslt = rbind(rslt, as_tibble(record()))
    }
    print(rslt)
}
    
simulate_once = function(){
    reset()
    gogo()
    return(record()$surv)
}

    • Solution III의 장점

      • 함수의 입력/출력을 상당부분 생략 가능, 함수의 출력을 어딘가에 저장하는 번거로운 과정도 스킵 가능, 구현이 쉽다.

    • Solution III의 단점

      • 지어된 변수들과 함수의 목록을 한번에 파악하기 어렵다.

        • 대안 : 모든 것을 하나로 묶는다.

      • 하나의 노트북에서 다른 풀이도 함께하다보면 변수 이름이 헷갈릴 수 있다.

        • 대안 : 이전 변수를 지우거나 커널 초기화를 활용한다.
